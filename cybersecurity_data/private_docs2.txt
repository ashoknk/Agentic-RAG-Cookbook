
### Paragraph 6: Fundamentals of Web Application Firewall Architectures and Deployment Models

At its core, a Web Application Firewall (WAF) operates as a security gateway positioned between external clients and web application origin servers to inspect and filter application-layer (Layer 7) traffic. Unlike traditional network firewalls that monitor transport-layer protocols and IP addresses, a WAF is protocol-aware, deeply analyzing HTTP and HTTPS request-response cycles. WAFs are commonly deployed in one of three architectural topologies: as a reverse proxy where all inbound traffic is routed through the firewall before reaching the application, as a transparent bridge/inline appliance, or via cloud-based edge delivery networks. By analyzing HTTP headers, query parameters, cookies, and POST bodies, the firewall enforces access control policies, decodes transfer encodings, and strips out anomalous payloads before they can interact with the underlying application logic.

### Paragraph 7: Taxonomy of Common Web Application Attacks and Vulnerabilities

Modern WAFs are primarily designed to defend against malicious exploits targeting application vulnerabilities, such as those codified in the OWASP Top 10. The most prevalent vectors include Injection flaws—such as SQL Injection (SQLi) and Command Injection—where an attacker embeds malicious database commands or OS directives into input fields, tricking the backend interpreter into executing unauthorized actions. Cross-Site Scripting (XSS) is another pervasive threat, where attackers inject malicious scripts into trusted websites to execute within an innocent user's browser session. Additionally, WAFs mitigate broken object-level authorization, path traversal attempts, and automated application-layer Denial of Service (Layer 7 DDoS) attacks, which seek to overwhelm web servers by generating a massive volume of resource-intensive HTTP GET or POST requests.
